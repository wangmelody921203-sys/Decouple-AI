# Week 11 Client Brief

## 五格框架

### 1) 我的客戶是誰
- 客戶名稱：Bailey
- 角色與背景：長期處於高壓與憂鬱狀態的上班族，對隱私高度敏感，不願向親友或同事揭露私事
- 當前階段：已有零散的音樂舒壓習慣，但缺乏可持續、可信賴且具隱私保護的情緒支持系統

### 2) 他表面上說想要什麼
- 明確要求：希望有一個不必向人交代私事，也能每天使用的情緒支持 Agent
- 交付期待：情緒陪聊、音樂安撫建議、每週情緒量表追蹤與風險提醒
- 時程／預算邊界：2 週做可運行驗證；先做個人版 MVP，再視效果擴展

### 3) 他真正想要的是什麼（最關鍵）
- Q1：他說出口的需求背後，真正擔心的是什麼？
	擔心情緒失控時沒有安全出口，且被他人知道後會被貼標籤。
- Q2：他沒說、但其實有的需求是什麼？
	他需要的是「被穩定接住」與「可被信任」：回應不能獵奇、不能說教、不能外洩，也不能在高風險情境給錯建議。
- Q3：如果我只交文件、不改決策能力，會失敗在哪裡？
	Bailey 會在最需要幫助時失去信任並停止使用，服務變成一次性展示而非可依賴支持。

### 4) 我們公司這次要做的核心承諾
- 這次最小可驗證成果（MVP）：
	交付可運行的情緒支持 Agent 原型（含安全回覆檢查、音樂建議、量表追蹤）。
- 可追蹤指標（成本、品質、可擴展性）：
	1) 高風險回覆誤判率下降 40% 以上
	2) 每週量表追蹤完成率達 80% 以上
	3) 平均回覆延遲維持在可即時安撫的區間（< 3 秒）
- 下輪可重用流程：
	每次交付前跑 /pulse-check，結束後跑 /memory-sync 回寫有效策略與錯誤修正。

### 5) Deliverable 形式（擇 3-5 項）
- [x] 可運行原型
- [x] 架構圖（含模型解耦）
- [x] 成本基線與優化對照
- [x] AI 輸出評判手冊
- [x] 7 日行動清單
- [x] Failure-to-Fix Log
