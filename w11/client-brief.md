# Week 11 Client Brief

## 五格框架

### 1) 我的客戶是誰
- 客戶名稱：WarmNest Living（暖巢家居）
- 角色與背景：營運負責人 Iris，帶 6 人電商團隊，主力販售寢具與家居品
- 當前階段：已上線 FAQ chatbot，但回覆品質不穩、客服工單仍高

### 2) 他表面上說想要什麼
- 明確要求：2 週內把客服 AI 回覆穩定下來，降低重工與錯誤承諾
- 交付期待：可運行原型、可操作的評判手冊、成本對照報告
- 時程／預算邊界：2 週驗證；先付基礎門票費，後續採節省成本分潤

### 3) 他真正想要的是什麼（最關鍵）
- Q1：他說出口的需求背後，真正擔心的是什麼？
	擔心旺季流量一上來客服失控，品牌評價下滑，退貨率升高。
- Q2：他沒說、但其實有的需求是什麼？
	他需要一套「團隊自己判斷 AI 回答能不能上線」的機制，而不是每次都等工程師救火。
- Q3：如果我只交文件、不改決策能力，會失敗在哪裡？
	團隊無法持續維護品質，系統一改版就回到靠人海補洞，客戶不會續聘。

### 4) 我們公司這次要做的核心承諾
- 這次最小可驗證成果（MVP）：
	交付可運行客服決策原型（含回覆前檢查節點）與模型解耦架構。
- 可追蹤指標（成本、品質、可擴展性）：
	1) 平均單輪對話 token 成本下降 25% 以上
	2) 高風險錯誤回覆率下降 40% 以上
	3) 併發量提升至現況 3 倍時仍可維持 SLA
- 下輪可重用流程：
	每次交付前跑 /pulse-check，結束後跑 /memory-sync 回寫有效策略與錯誤修正。

### 5) Deliverable 形式（擇 3-5 項）
- [x] 可運行原型
- [x] 架構圖（含模型解耦）
- [x] 成本基線與優化對照
- [x] AI 輸出評判手冊
- [x] 7 日行動清單
- [x] Failure-to-Fix Log
