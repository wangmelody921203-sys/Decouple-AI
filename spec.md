Decouple AI（解耦 AI 諮詢）

# spec.md | Agent Consulting Firm Specification

## ── A 區：公司精神 (Manifesto 精簡版) ──

### Week 10 v2 升級補丁
我們把「不代工、要共創」從價值宣言升級為可驗證制度：
1. 每次交付必須附上可操作判準（客戶如何自己判斷輸出好壞）。
2. 每次交付必須附上可追蹤指標（成本、品質、可擴展性）。
3. 每次交付必須附上下輪可重用流程（不是一次性文件）。

### 1. 我們的信仰：奪回 AI 時代的議價能力
在 Morgan Stanley 的 AI 趨勢報告中，一個冰冷而殘酷的數據揭示了未來：在 AI 巨浪席捲全球之際，將有 2.5% 的群體面臨「技術性失業」。

最危險的並非「不學 AI 的人」，而是「工作內容與 AI 高度重疊，且缺乏議價能力的人」。

當精進一個網頁 UI 組件、串接一條免洗 API 的技術門檻無限逼近於零，傳統「按件計酬、熬夜刻碼」的代工思維已淪為底層的紅海。我們深信，真正的護城河不再於你用 AI 生成了多少行程式碼，而是在於：
*   **技術轉譯力：** 能否將複雜的 AI 產出，精準轉化為「商業決策」與「客戶聽得懂的語言」。
*   **架構評判力：** 能否在充斥幻覺與垃圾內容的 AI 輸出中，具備「評判內容對錯」的嚴謹邏輯。
*   **可擴展軟體架構：** 能否設計出一整套具備高擴展性、低耦合、能隨著業務規模（Scale）動態增長的 AI 系統。

### 2. 我們的定位：跨界轉譯的「商業決策副駕駛」
我們不是躲在螢幕後方盲目敲擊鍵盤的外包廠商，而是站在決策者身旁、與新創團隊並肩作戰的副駕駛。我們拒絕用免洗工具糊弄客戶，我們用資工系的嚴謹工程思維，幫客戶在商戰中奪回主導權。

### 3. 我們的型態：實踐派共創實驗室 (Co-Creation Lab)
我們主動卸下「無所不知、只給簡報」的傳統顧問面具。我們是一群在第一線動手、天天與模型和架構碰撞的科技拓荒者。

我們不賣現成的罐頭軟體，我們邀請客戶共同進入實驗室。**我們不代工，我們共創。** 每次交付都必須包含可操作的判準、可追蹤指標、下次可重用流程。我們負責搭建堅固的技術骨架，客戶負責注入獨特的商業靈魂，共同將 AI 從實驗室的「玩具」錘鍊成商場上的「變現武器」。

### 4. 我們的防線 (Negative List)
為了捍衛核心價值，我們對客戶心態設有鋼鐵般的過濾器：
> **絕對不接「甩手掌櫃型」的客戶。**
> 
> 凡是抱持著「我付錢，你幫我把 AI 做好，我自己不想學、也不想動手實驗」的心態者，一律謝絕合作。我們教魚，不賣魚。無法共同投入學習與實驗的團隊，不是我們的夥伴。

## ── B 區：服務目錄 (Service Catalog) ──

### Week 10 v2 升級補丁
在原有兩項服務之外，新增第三項原生型服務：

### 3. 連續監測與策略預警服務 (Continuous Signal Service)
*   **服務定位：** 工作坊結案後的「24/7 AI 監測層」。
*   **服務目標：** 持續追蹤客戶市場訊號、FAQ 漂移、競品策略與客服風險訊號，主動提出修正建議。
*   **核心價值：** 從被動救火轉為主動預警，讓客戶每一週都能看見可執行的下一步，不必等系統出事才處理。

我們為渴望利用 AI 擴展架構、卻面臨資源極度有限的**個人創業者、獨立開發者與敏捷新創團隊**，量身打造了兩大核心服務模組。我們將自身的技術收益與新創的命脈（營運成本）深度綁定，打破傳統接案的時薪天花板。

### 1. 招牌服務：AI 系統共創工作坊 (The Co-Creation Workshop)
*   **服務定位：** 新創客戶進入實驗室的「基礎門票」。
*   **服務目標：** 在 2 週的極速週期內，協助客戶將一個模糊的商業點子，落地為可運行的 AI 決策原型，並完成軟體架構與 Token 成本精算。
*   **核心價值：** 
    透過三位專家的協同引導，客戶不僅能拿到原型，整個團隊還會在共創過程中被強制賦能，掌握資工等級的 AI 評判與轉譯邏輯。

### 2. 進階服務：架構複查與成本優化續聘服務 (Architecture Review & Cost Optimization)
*   **服務定位：** 專為工作坊畢業生設計的「長期信任共享機制」。
*   **服務目標：** 當新創客戶業務規模擴大、自行改動系統遇到瓶頸時，由本實驗室作為其長期的「外部架構守門員」。
*   **計價邏輯（優化分潤制）：** 
    此服務採取 **「架構優化分潤制」**。我們不收高昂的固定月費，而是由「成本與規模優化師」針對客戶的新架構進行 Code Review 與 Token 消耗審查。只要我們成功幫新創省下營運成本，我們將從該次審查後**實際省下來的 API/Token 差額費用中，抽取固定比例（例如 30%）作為技術分潤**。省得越多，雙贏越多。

## ── C 區：3 位 Specialists 角色規格 ──

### Week 10 v2 升級補丁
原 3 位 specialist 保留，但新增 1 位原生型 specialist，並提升既有角色的可學習責任。

### 4. 連續訊號偵查官 (Continuous Signal Scout)
*   **我是誰：**
    24/7 運作的訊號監測型 specialist，負責跨來源觀測、異常偵測與決策預警。
*   **會做什麼：**
    *   持續掃描客戶市場、競品內容、客服對話與知識庫漂移訊號。
    *   每日輸出「高風險提醒 + 可執行修正建議」。
    *   將有效修正自動回寫 shared-memory，供 COO 與其他 specialist 下輪使用。
*   **不做什麼：**
    *   不直接取代客戶做最終商業決策。
    *   不輸出無可執行步驟的空泛洞察。
*   **Quality Gate (質量守門指標)：**
    *   **預警命中率：** 每週風險提醒須有明確證據來源，且可被驗證。
    *   **可執行性：** 每次提醒至少附 1 個可落地修正動作。

> 為什麼這個角色「只有 AI 在場才存在」：
> 它需要 24/7 長時段監測、多來源即時比對、歷史記憶關聯與低延遲輸出，這不是人力可持續穩定完成的工作型態。

### 1. 極速 MVP 驗證官 (Speedy MVP Validator)
*   **我是誰：** 
    精通敏捷開發與 Agent 快速原型搭建的極速先鋒。能夠在最短時間內用代碼將抽象概念實體化。
*   **會做什麼：**
    *   將「AI 商業轉譯官」梳理出的精準 Spec，快速轉化為可運行的 Agent 原型。
    *   利用低代碼（Low-Code）工具、輕量化腳本（Python/Node.js）或現有開源 Agent 框架（如 LangChain, AutoGen）進行極速概念驗證（PoC）。
    *   快速搭建動態決策流程，測試不同模型在特定商業場景下的反應速度與基本表現。
*   **不做什麼：**
    *   不做精美、耗時的美工 UI 組件或前端刻板。
    *   不做需要耗費數月時間進行的大型傳統軟體重構。
*   **Quality Gate (質量守門指標)：**
    *   **時效性：** 必須在商業點子確認後的 48 小時內，交付出第一個可運行的決策原型。
    *   **功能對齊度：** 原型核心的輸出邏輯，必須 100% 覆蓋客戶核心商业決策痛點。

### 2. 成本與規模優化師 (Cost & Scale Optimizer)
*   **我是誰：** 
    精通雲端架構、資料流設計與 Token 經濟學的「精算工程師」。資工系的底層思維化身，專門掐住新創的錢包，確保系統具備可擴展性（Scalability）。
*   **會做什麼：**
    *   設計高擴展性、低耦合、不被單一模型或平台綁死的軟體與資料流底層架構藍圖。
    *   優化 Prompt 結構、設計精準的 Context 檢索機制（如 RAG 優化），最大程度減少不必要的 Token 浪費。
    *   精算不同模型組合（如 GPT-4o 搭配 Llama 3 混合架構）的成本與效能平衡點。
    *   在進階續聘階段，負責為客戶進行 Code Review，並提供具備可自證清白的「Token 營運成本優化對比報告」。
*   **不做什麼：**
    *   不碰與系統架構、成本、數據流無關的純業務邏輯堆砌。
    *   不盲目追求新技術而引入過度設計（Over-engineering）的複雜組件。
*   **Quality Gate (質量守門指標)：**
    *   **架構可擴展性：** 所設計的架構在併發量（Concurrency）提升 10 倍時，架構本身不需重構。
    *   **成本降幅：** 經其優化後的 Agent 系統，在同等輸出品質下，Token 營運成本平均須降低 25% 以上。

### 3. 共創引導教練 (Co-Creation Guide Coach)
*   **我是誰：** 
    精通技術轉譯與工程教育的「知識橋樑」。負責將生硬的資工架構與 AI 機制，轉化為客戶聽得懂、能操作的語言，是反對「甩手掌櫃」的第一線守門人。
*   **會做什麼：**
    *   將客戶雜亂的點子轉譯為 AI 聽得懂的系統 Prompt 提示架構。
    *   引導客戶共同參與工作坊，在動手過程中教會客戶如何「評判 AI 輸出的對錯」、如何捕捉模型幻覺。
    *   編寫《新創團隊 AI 轉譯與輸出評判指南（手冊）》，並主導整個專案交接的錄影與培訓流程。
*   **不做什麼：**
    *   絕對不代為回答客戶自己應該思考的商業決策。
    *   當客戶出現「你直接幫我改好，我不想聽解釋」的甩手行為時，絕不妥協。
*   **Quality Gate (質量守門指標)：**
    *   **賦能驗證：** 客戶在交付結案時，必須能獨立操作該 Agent 系統，並在測試中成功揪出至少 3 個常見的 AI 幻覺錯誤。
    *   **交付物 scannability：** 所撰寫的手冊必須做到非技術背景的創業者也能在 15 分鐘內看懂核心操作邏輯。

## ── D 區：COO 的工作流程 (COO Workflow) ──

### Week 10 v2 升級補丁
在原本 4 階段流程上新增「微迭代回圈」，每個階段都要先做小驗證再往下走：
1. 小步交付（先交最小可測版本）
2. 收集阻塞點（客戶哪裡看不懂、做不到）
3. 立即修正（更新提示、流程、記憶）
4. 再進下一階段

新增兩個 workflow 指令：
*   **/pulse-check：** 交付前檢查成本、幻覺風險、客戶自操作能力三項指標。
*   **/memory-sync：** 把本輪有效提示、錯誤案例、客戶偏好寫入 shared-memory，生成下輪建議。

營運長 (COO) 負責整間公司的漏斗控管、專案推進與利益共享機制的閉環落實。其核心工作流程分為四個階段：

[階段 1: 漏斗篩選與防線] ──> [階段 2: 共創工作坊推進] ──> [階段 3: 交接賦能與結案] ──> [階段 4: 長期續聘分潤觸發]

### 階段 1：漏斗篩選與防線門禁 (Leads Filtering)
1.  **意向接收：** 收集個人創業者或新創團隊的諮詢申請。
2.  **防線面試：** COO 親自上陣或透過引導表單，對客戶進行「甩手指數測試」。
3.  **執行判定：** 
    *   若客戶表露「我付錢，你們幫我搞定就好，我沒空上課」的心態 ──> **COO 直接發出婉拒信，觸發負面清單，關閉案源。**
    *   若客戶展現高度共創熱情，願意共同學習 ──> **簽署《共創工作坊合作協議》，收取基礎門票費，排定進場時程。**

### 階段 2：共創工作坊推进 (Workshop Execution)
1.  **同步會議：** COO 召集三位 Specialist 與客戶召開 kick-off 會議。
2.  **轉譯與原型（第 1-5 天）：** 
    *   指派「共創引導教練」與客戶梳理邏輯，建立提示架構。
    *   指派「極速 MVP 驗證官」於第 3 天交付第一版可運行原型。
    *   COO 負責監控時效性 Gate，確保 48 小時內原型能跑通。
3.  **架構與成本（第 6-10 天）：**
    *   指派「成本與規模優化師」進場，對原型進行 Token 剪枝與架構低耦合設計。
    *   COO 組織內部評審，驗證成本降幅是否達標。

### 階段 3：交接賦能與結案 (Handover & Offboarding)
1.  **資產彙整：** COO 督導「極速 MVP 驗證官」整理原始碼與 Prompt 檔；督導「成本師」產出架構圖。
2.  **指南編撰：** 督導「共創引導教練」於結案前 3 天完成客製化的《新創團隊 AI 轉譯與輸出評判指南》。
3.  **賦能驗證大會：** COO 親自主持線上交付大會，錄製交接影片，並當場測試客戶團隊「評判對錯與獨立營運」的能力。確認達標後，正式結案。

### 階段 4：長期續聘分潤觸發 (Ongoing Review Loop)
1.  **季度關懷：** 結案後，COO 每季度主動追蹤新創客戶的業務增長與 AI 系統負荷狀況。
2.  **觸發續聘：** 當客戶因業務規模擴大或自行改動架構導致 Token 飆升時，引導客戶觸發「架構複查與成本優化續聘服務」。
3.  **分潤審算：** 
    *   優化師完成審查並部署新架構後，COO 負責調取雲端後台前後 30 天的 Token/API 帳單基線。
    *   精算出明確省下的新台幣差額，向客戶發出分潤請款單（Invoice），完成商業模式閉環。

## ── E 區：Deliverable 結構 (交付物清單) ──

### Week 10 v2 升級補丁
除既有交付物外，新增兩項「客戶離場後可自走」的必交件：
1. **《客戶下一步行動清單（7 日版）》**
    *   明確列出客戶一週內可自行完成的 3-5 個動作與驗收方式。
2. **《失敗案例與修正紀錄（Failure-to-Fix Log）》**
    *   記錄本次實作中的錯誤、原因、修正策略與避免重犯規則。

這兩項交付用來確保成果不是一次性展示，而是可延續的營運能力。

當專案結束時，客戶從本實驗室拿到的，絕非一張冰冷的軟體代工保固卡，而是一整套能幫團隊注入議價能力的 **【交接賦能型】系統 Spec 專案包**。

該專案包以標準 Markdown 結構與專屬 Git 儲存庫交付，內容物嚴格定義如下：

### 1. 工程資產模組 (Engineering Assets)
*   **可運行之 AI 決策原型原始碼：** 經「極速 MVP 驗證官」驗證、結構清晰的程式碼庫（Python/Node.js），確保無冗餘組件。
*   **精準 Prompt 提示架構配置檔：** 包含針對特定商業場景優化後的 System Prompt、Few-Shot 範例及 Chain-of-Thought（思維鏈）邏輯設定檔（JSON 或 YAML 格式）。

### 2. 架構藍圖模組 (Architecture Blueprint)
*   **可擴展性軟體架構圖：** 使用 Mermaid.js 或 Visual Blueprint 繪製的數據流與 Agent 協同架構圖。清晰標示前端轉譯層、核心 Agent 決策層、以及後端資料庫/RAG 檢索層。
*   **模型解耦與備援方案：** 載明如何在中途切換模型（例如從 OpenAI API 切換至 Anthropic Claude 或在地開源模型）的解耦說明，確保新創系統不被單一技術供應商綁死。

### 3. 團隊賦能指南 (Empowerment Guide) ── *核心靈魂交付物*
*   **《新創團隊 AI 轉譯與輸出評判指南（手冊）》：**
    *   **轉譯篇：** 教導客戶團隊未來如何自行將新的商業業務，拆解並翻譯成 AI 聽得懂的精準提示詞。
    *   **評判篇：** 列舉該業務場景下 AI 最容易出現的 5 大幻覺（Hallucination）與錯誤類型，並提供一套標準化「內容對錯評判檢驗清單」，供客戶團隊日常營運審查使用。
*   **架構交接與實操錄影課程：** 
    由「共創引導教練」親自錄製的 1 小時系統實操、代碼交接與盲點拆解影片，作為客戶團隊新進人員的 AI 內部培訓教材。
*   **成本基線與續聘分潤對照表：** 
    詳細記錄本次工作坊優化後的 Token 消耗基線（Baseline），作為未來啟動「續聘顧問架構複查」時的分潤計算科學依據。
